Инпут с автодополнением

Дата атрибуты:

  • data-autocomplete - отвечает за реализацию модуля
  • data-autocomplete-callback="name" - коллбек получения данных, при завершии должен вызывать кастомное событие 'loadDataSuccess', задать свой коллбек можно в init-autocomplete.js в объекте callbacks, без аттрибута вызывается базовый коллбек.
  • data-multi-options - реализует мультивыбор элементов, добавляет кнопку по которой можно вызвать список выбранных элементов, при его отсутствии данные вставляются в инпут.
  • data-error-msg="сообщение" - для каждого экземпляра можно установить разное сообщение о результатах поиска, после инициализации аттрибут удаляется

Инпут с мультивыбором c data-multi-options

Пример вызова

+custom-input({inputLabel: "Введите что-что", inputType: "text", inputPlaceholder: "Название локации", inputError: "", inputName: "", inputLoader: true, inputAutocomplete: 'off'})(data-validate-type="text", data-autocomplete, data-multi-options)

Инпут с вставкой выбранного элемента в инпут без data-multi-options

Пример вызова

+custom-input({inputLabel: "Введите что-что", inputType: "text", inputPlaceholder: "Название локации", inputError: "", inputName: "", inputLoader: true, inputAutocomplete: 'off'})(data-validate-type="text", data-autocomplete)

Элемент с классом autocomplete-menu генерируется через js при успешной загрузке по событию 'loadDataSuccess', экземпляры классов вынесены в массив window.autocomplete
Чтобы проверить работу введите в инпут любую латинскую букву.
Для показа списка предлагаемых элементов, элементу autocomplete-menu добавляется класс .is-active-options
Для показа списка выбранных элементов, элементу autocomplete-menu добавляется класс .is-active-selected
Для показа ошибки при отсутсвии при совпадении, элементу autocomplete-menu добавляется класс .is-missing
Для показа лоадера, элементу custom-input добавляется класс .custom-input--has-loader
Для показа кнопки с выбранными элементами, элементу custom-input добавляется класс .custom-input--has-counter-btn
